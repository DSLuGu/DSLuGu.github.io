### Surveys 테이블 전처리
    * answer 값이 't', 'f'로 입력되어 있는데 이를 True, False로 변환
    * created_at과 updated_id을 datetime 형식으로 변환
    * 같은 user_id가 여러번의 설문조사를 응답한 경우 제거


```python
# 새로운 column을 만들어 answer의 t 값과 f값을 True, False로 변환
for i in range(1, 13):
    old_column = f'answer{i}_selected'
    new_column = f'answer{i}_selected(bool)'
    surveys[new_column] = (surveys[old_column] == 't')

# created_at과 updated_at을 datetime 자료형으로 변환
surveys["created_at"] = pd.to_datetime(surveys["created_at"])
surveys["updated_at"] = pd.to_datetime(surveys["updated_at"])

# 같은 유저가 여러번 설문조사를 응답한 경우를 제거
surveys = surveys.drop_duplicates(['user_id', 'question_id'], keep='last')
```

### Payments 테이블 전처리
    * 모든 연락처의 양식을 010-xxxx-xxxx으로 통일
    * 신청 수업을 쪼개어 수업타입과 기수를 만들어내는 작업
    * '신청날짜' column을 datetime 형식으로 변환
    * 상태가 결제 완료인 경우는 True, 아닌 경우는 False
    * 신청날짜 칼럼을 이용하여 월, 일, 요일, 시간 column을 추가
    * 문자 자료형인 금액을 ,를 제거한 뒤 숫자 자료형(int)로 생성


```python
payments = pd.read_sql("SELECT * FROM 'payments'", connect)
print('Payments 테이블 크기: ' ,payments.shape)


valid_phone_number = (payments["연락처"].str.contains("^010|\+82"))
payments["연락처(clean)"] = payments["연락처"]
payments.loc[~valid_phone_number, "연락처(clean)"] = pd.np.nan

def clean_phone_number(phone_number):
    if pd.isnull(phone_number):
        return pd.np.nan

    # 대시가 문자열에 이미 있다면 번호를 그대로 반환
    if "-" in phone_number:
        return phone_number
    
    # 대시가 문자열에 없다면 자릿수에 맞추어 대시 추가
    else:
        phone_number = phone_number[0:3] + "-" + phone_number[3:7] + '-' + phone_number[7:]
        
        return phone_number

# 연락처 데이터에 clean_phone_number 함수를 각각 적용
# 그리고 이 데이터들로 연락처(clean) column을 대체
payments["연락처(clean)"] = payments["연락처(clean)"].apply(clean_phone_number)
payments[["연락처", "연락처(clean)"]].head()
```

### 홈페이지에 처음 접근한 순간부터 결제까지 걸린 시간 계산

> 대부분의 사용자가 홈페이지에 접속하자마자 결제를 하는 것이 아닌, 충분한 고민 후에 결제를 진행합니다. 사람들이 결제를 하는데까지 고민하는 시간을 잘 정리한다면 마케팅 부서에서 리타게팅 전략을 펼칠 때 도움이 될 것으로 생각됩니다.


```python
# created_at은 세션이 생성된 시간이고 '신청날짜'는 실제로 수업을 신청한 시간
time = data['신청날짜'] - data['created_at']
time_int = time.dt.components['days'] + time.dt.components['hours'] / 24 \
        + time.dt.components['minutes'] / (24*60) + time.dt.components['seconds'] / (24 * 60 * 60)

# time_int 데이터를 이용해 처음 접속 후 10일 이내 결제한 사람은 몇% 인가? 를 쉽게 파악할 수 있는 그래프 생성
# 그래프를 그리기 위한 라이브러리
import seaborn as sns
sns.set_style("whitegrid", {'grid.linestyle': '--'})

plt.figure(figsize=[18,8])
ax = sns.distplot(time_int,
                  hist=False,
                  kde_kws={'cumulative' : True})

ax.set_xlim(0, 25)
```

> 위의 분포를 통해 결제를 한 사람들 중 80% 이상이 10일 이내 결제, 그리고 나머지 20%는 더욱 오랜 기간동안 고민을 한다는 것을 파악할 수 있습니다. 이를 통해 관심을 가졌지만 아직 결제를 하지 않은 잠재고객들의 결제 가능성을 역으로 추정해 리타겟팅 전략의 예산 분배를 진행할 수 있습니다.
